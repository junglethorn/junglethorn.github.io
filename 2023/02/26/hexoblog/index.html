有一点也是github的更新问题：默认分支从以前的master换成了main。这是由于一些米国人的racial问题引发的，间接导致了计算机小白受害。因此，我们需要解决main分支和master分支的冲突。

一种方法是删去其中一个，并把上面的那串代码的branch改成相应的main/branch。

或者在仓库的setting中，修改默认分支，还要在page处确认是自己在使用的分支。

个性主题配置

本人以使用fluid主题为例

用户手册：https://hexo.fluid-dev.com/docs/start/

在本地theme文件夹中加入下载好的fluid主题，将其文件夹名复制，在外面的config_yml文件中的倒数第二段“theme“改成文件夹名（可能是hexo-theme-fluid或fluid）

之后的大部分主题配置在fluid文件夹的config_yml中配置修改，详见上述用户手册。

hexo基础用法

若执行hexo g出错则执行npm install hexo --save
若执行hexo d出错则执行npm install hexo-deployer-git --save
错误修正后再次执行hexo g和hexo d上传到服务器
